# Модуль бесшовной интеграции внешнего контента

Интеграционный модуль на базе технологии **Web Components (Shadow DOM)**. Предназначен для плавного, изолированного и безопасного запуска внешних партнёрских приложений внутри текущей страницы пользователя.

## Архитектура решения
Решение состоит из трёх полностью изолированных слоев, каждый из которых работает как независимый блок:
1. **Сайт-клиент:** Основной веб-ресурс, который инициирует сессию, динамически добавляя кастомный HTML-элемент на страницу при нажатии кнопки «Запустить игру».
2. **Модуль-интегратор (`<game-module>`):** Инкапсулированный компонент, управляющий жизненным циклом сессии, изоляцией стилей, блокировкой интерфейса основного сайта и логикой кроссфейда между лоадером и ифреймом.
3. **Внешнее целевое приложение (Игра):** Автономное приложение, работающее внутри изолированного `iframe`. Оно не имеет прямого доступа к DOM хост-страницы и сообщает системе о готовности или выходе через сообщения окон.

## Инструкция по запуску
Проект состоит из трёх независимых приложений:

- `site` - основной сайт-клиент, порт:http://localhost:5173
- `game-module` - интеграционный Web Component, порт:http://localhost:5174
- `game-app` - внешнее приложение, порт: http://localhost:5175
Каждое приложение запускается отдельно с помощью команды npm run dev

## Протокол взаимодействия
Связь между интеграционным модулем и внешним приложением происходит через механизм `window.postMessage()`.
Внешнее приложение загружается внутри изолированного `iframe` и не имеет прямого доступа к DOM основного сайта. Все события между интеграционным модулем и внешним приложением передаются через сообщения окон.

### Событие READY
После полной загрузки внешнее приложение отправляет сообщение интеграционному модулю:
```javascript
window.parent.postMessage({
    type: 'READY'
}, '*')
```
После получения события `READY` интеграционный модуль:
- скрывает loader;
- делает внешний `iframe` видимым;
- передает управление внешнему приложению.

### Событие CLOSE
При завершении работы внешнее приложение отправляет сообщение:
```javascript
window.parent.postMessage({
    type: 'CLOSE'
}, '*')
```
После получения события `CLOSE` интеграционный модуль:
- удаляет себя из DOM;
- очищает глобальные обработчики событий;
- снимает блокировку основной страницы;
- восстанавливает скролл и исходное состояние интерфейса.

## Пример подключения модуля на страницу
С помощью использования Web Components, модуль бесшовно интегрируется в любую frontend-экосистему (Vanilla JS, React, Vue, Angular и др.) без необходимости сложной настройки сборщиков со стороны разработчиков-клиентов.

```html
<!-- 1. Подключаем скрипт модуля-интегратора -->
<script type="module" src="./src/game-module.js"></script>

<!-- 2. Используем кастомный тег в HTML или создаем динамически через JavaScript -->
<game-module src="http://localhost:5175/"></game-module>
```

## Проверка сценария
1. Открыть site
2. Нажать "Запустить игру"
3. Проверить:
   - появился overlay
   - появился loader
   - после READY загрузилась игра
   - кнопка "Выйти" закрывает модуль
   - скролл страницы восстановился
